Decorators

Decorations are compile time annotations on functions, types, global constants and variables. Similar to Java annotations, a decoration may also take arguments. A decorator can also represent a bundle of decorators.

New decorators

The basic syntax for creating a new decorator is decorator <declaration type> @<name>. The type category is one of var, const, func, union, struct, enum, typedef and error.

For example:

decorator func @mydecorator;

func void foo() @mydecorator
{ ...  }

Multi type decorators

It's possible to make a decorator match more than a single declaration type, by separating them using comma:

decorator func, enum, struct @foodecorator;

func void foo() @foodecorator
{ 
    /* ... */  
}

struct Bar @foodecorator
{ 
    int i;
}

Arguments

A decorator may take a number of arguments, similar to a function. All such arguments must be compile time constants.

decorator func @intdec(int i)

void foo() @intdec(2)
{ ... }

Just like for function calls, the arguments may have defaults, use named arguments etc. However, varargs are not allowed.

decorator func @dec(int d, int i = 0, float f = 1.0)

void foo() @dec(2)
{ ... }

void bar() @dec(7, f = 3.0)
{ ... }

Compile time decorator access

What's useful about decorators is that they can be accessed during compile time:

macro @fooCheck($a)
{
    $if (@defined($a.@foo))
    {
        return "Was fooed";
    }
    return "Ok";
}

struct TestA
{
    int i;
}

struct TestB @Foo
{
    float f;
}

func void test()
{
    printf("Check TestA: " @fooCheck(TestA) "\n");    
    printf("Check TestB: " @fooCheck(TestB) "\n");
    // Prints:
    // Check TestA: Ok
    // Check TestB: Was fooed
}

The values may be recovered:

It's also possible to get hold of the values:

decorator func @myvar(int i = 0);

func void test() @myvar(200) 
{ 
    /* ... */
}

func void test()
{
    printf("%d", test.@myvar.i); // Prints 200
}

Bundled decorators

It's also possible to create bundles of decorators, that simply contain other decorators. Unlike a normal decorator, a decorator bundle is a simple alias, so it has no real compile time information.

decorator func @foodec;
decorator func @bardec(int i = 0);

decorator @mixdec alias @foodec, @bardec(3);

// As if we had added @foodec, @bardec(3)
func void test() @mixdec
{
    /* ... */
}

A decorator bundle might also be completely empty.

Iterating over decorators

Within a macro it's possible to iterate in compile time over all non-alias decorators using $each.

decorator struct @special;

struct TestA @special { int i; }
struct TestB { float f; }
struct TestC @special { float f; }

macro @specialStructs()
{
    string[+] res = {};
    $each(@special as $x)
    {
        res += @name($x);
    }
    // The above expands to:
    // res += "TestA";
    // res += "TestC";    
    return res;
}

// SPECIAL_STRUCTS = { "TestA", "TestB" }
const string[] SPECIAL_STRUCTS = @specialStructs();
